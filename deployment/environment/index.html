    Environment Setup¶

    This guide covers setting up Aspectify in different environments for development, testing, and production.

    Development Environment¶

    Local Development Setup¶

    1. Clone Repository 

      git clone https://github.com/your-username/aspectify.git
cd aspectify

    2. Create Virtual Environment 

      python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate

    3. Install Dependencies 

      pip install -r requirements.txt

    4. Environment Configuration 

      # Copy example environment file
cp .env.example .env

# Edit .env file with your settings
GOOGLE_API_KEY=your_google_api_key_here
LOG_LEVEL=DEBUG

    5. Verify Installation 

      python main.py

    Development Tools¶

    Testing Environment¶

    Test Configuration¶

    1. Test Environment File (.env.testing): 

      GOOGLE_API_KEY=test_api_key
LOG_LEVEL=DEBUG
PROJECT_NAME=Aspectify-Test

    2. Test Database (if using database): 

      DATABASE_URL=postgresql+asyncpg://test:test@localhost:5432/aspectify_test

    Running Tests¶

    # Run all tests
pytest

# Run with coverage
pytest --cov=app --cov-report=html

# Run specific test file
pytest tests/unit/test_absa_agent.py

# Run with verbose output
pytest -v

    Production Environment¶

    Production Configuration¶

    1. Production Environment File (.env.production): 

      GOOGLE_API_KEY=production_api_key
LOG_LEVEL=INFO
PROJECT_NAME=Aspectify
PROJECT_VERSION=1.0.0

    2. Security Settings: 

      # Use strong, unique API keys
GOOGLE_API_KEY=your_production_google_api_key

# Set appropriate log level
LOG_LEVEL=INFO

# Use production database
DATABASE_URL=postgresql+asyncpg://user:password@prod-db:5432/aspectify

    Production Deployment¶

    Docker Deployment¶

    1. Dockerfile: 

      FROM python:3.11-slim

WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# Copy requirements and install Python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY . .

# Create logs directory
RUN mkdir -p logs

# Set environment variables
ENV PYTHONPATH=/app
ENV LOG_LEVEL=INFO

# Run application
CMD ["python", "main.py"]

    2. Docker Compose (docker-compose.yml): 

      version: '3.8'

services:
  aspectify:
    build: .
    ports:
      - "8000:8000"
    environment:
      - GOOGLE_API_KEY=${GOOGLE_API_KEY}
      - LOG_LEVEL=INFO
    volumes:
      - ./logs:/app/logs
    restart: unless-stopped

    3. Deploy with Docker: 

      # Build and run
docker-compose up -d

# View logs
docker-compose logs -f aspectify

    Environment-Specific Configurations¶

    Development¶

    • Log Level: DEBUG for detailed information
    • API Keys: Use development/test keys
    • Database: Local development database
    • Error Handling: Verbose error messages

    Testing¶

    • Log Level: DEBUG for test debugging
    • API Keys: Use test API keys
    • Database: Test database with sample data
    • Error Handling: Detailed error reporting

    Production¶

    • Log Level: INFO for normal operation
    • API Keys: Production API keys with proper permissions
    • Database: Production database with backups
    • Error Handling: Secure error messages, no sensitive data

    Monitoring and Logging¶

    Production Logging¶

    1. Log Rotation: 

      # Set up logrotate
sudo nano /etc/logrotate.d/aspectify

# Content:
/var/log/aspectify/*.log {
    daily
    missingok
    rotate 52
    compress
    delaycompress
    notifempty
    create 644 aspectify aspectify
}

    2. Log Monitoring: 

      # Monitor logs in real-time
tail -f logs/app.log

# Search for errors
grep "ERROR" logs/app.log

# Count log entries by level
grep -c "INFO" logs/app.log

    Health Checks¶

    Create a health check endpoint:

    # health_check.py
import asyncio
from app.core.config import settings
from app.core.logging_config import logger

async def health_check():
    try:
        # Check configuration
        if not settings.GOOGLE_API_KEY:
            return {"status": "unhealthy", "error": "Missing API key"}

        # Check logging
        logger.info("Health check performed")

        return {"status": "healthy", "timestamp": "2025-10-12T11:00:00Z"}
    except Exception as e:
        logger.error(f"Health check failed: {e}")
        return {"status": "unhealthy", "error": str(e)}

if __name__ == "__main__":
    result = asyncio.run(health_check())
    print(result)

    Troubleshooting¶

    Common Issues¶

    1. API Key Not Found: 

      # Check environment variables
echo $GOOGLE_API_KEY

# Check .env file
cat .env

    2. Permission Errors: 

      # Fix log directory permissions
chmod 755 logs/
chmod 644 logs/app.log

    3. Memory Issues: 

      # Monitor memory usage
free -h
ps aux --sort=-%mem | head

    4. Network Issues: 

      # Test API connectivity
curl -H "Authorization: Bearer $GOOGLE_API_KEY" \
     https://generativelanguage.googleapis.com/v1beta/models

    Performance Optimization¶

    1. Resource Limits: 

      # Set memory limits
ulimit -v 2097152  # 2GB virtual memory

# Set file descriptor limits
ulimit -n 4096

    2. Process Management: 

      # Use process manager like PM2
npm install -g pm2
pm2 start main.py --name aspectify
pm2 monit

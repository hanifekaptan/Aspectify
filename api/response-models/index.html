    Response Models¶

    Aspectify uses Pydantic models for structured data validation and serialization.

    ABSAResponse¶

    Main response model for aspect-based sentiment analysis results.

    Schema¶

    class ABSAResponse(BaseModel):
    id: str
    polarity: str
    score: float
    aspects: List[AspectAnalysisResponse]
    message: str
    text: str

    Fields¶

    Field Type Description
    id str Unique identifier (UUID) for the analysis
    polarity str Overall sentiment: "positive", "negative", or "neutral"
    score float Confidence score (0.0-1.0) for overall sentiment
    aspects List[AspectAnalysisResponse] List of individual aspect analyses
    message str Status message indicating success or failure
    text str Original text or transcribed audio content

    Example¶

    {
  "id": "123e4567-e89b-12d3-a456-426614174000",
  "polarity": "neutral",
  "score": 0.5,
  "aspects": [
    {
      "aspect": "camera quality",
      "sentiment": "positive",
      "score": 0.95,
      "quote": "camera quality is excellent"
    },
    {
      "aspect": "battery life",
      "sentiment": "negative",
      "score": 0.98,
      "quote": "battery life is disappointing"
    }
  ],
  "message": "ABSA completed successfully.",
  "text": "The camera quality is excellent but the battery life is disappointing."
}

    AspectAnalysisResponse¶

    Individual aspect analysis within the main response.

    Schema¶

    class AspectAnalysisResponse(BaseModel):
    aspect: str
    sentiment: str
    score: float
    quote: str

    Fields¶

    Field Type Description
    aspect str Name of the aspect (e.g., "camera", "battery", "service")
    sentiment str Sentiment for this aspect: "positive", "negative", or "neutral"
    score float Confidence score (0.0-1.0) for this aspect's sentiment
    quote str Relevant text excerpt that supports this aspect analysis

    Example¶

    {
  "aspect": "camera quality",
  "sentiment": "positive",
  "score": 0.95,
  "quote": "camera quality is excellent"
}

    LLMResponse¶

    Response model for raw LLM analysis results.

    Schema¶

    class LLMResponse(BaseModel):
    polarity: str
    score: float
    aspects: List[AspectAnalysisResponse]

    Fields¶

    Field Type Description
    polarity str Overall sentiment from LLM
    score float Overall confidence score
    aspects List[AspectAnalysisResponse] Aspect analyses from LLM

    Data Validation¶

    All response models use Pydantic for automatic validation:

    Type Validation¶

    # Automatic type conversion and validation
response = ABSAResponse(
    id="123",  # str
    polarity="positive",  # str
    score=0.95,  # float
    aspects=[],  # List[AspectAnalysisResponse]
    message="Success",  # str
    text="Sample text"  # str
)

    Validation Errors¶

    from pydantic import ValidationError

try:
    response = ABSAResponse(
        id="123",
        polarity="invalid",  # Not in allowed values
        score="not_a_number",  # Not a float
        aspects=[],
        message="Success",
        text="Sample text"
    )
except ValidationError as e:
    print(e.errors())

    Serialization¶

    JSON Serialization¶

    # Convert to JSON
json_data = response.model_dump_json(indent=2)

# Convert to dictionary
dict_data = response.model_dump()

    JSON Deserialization¶

    # From JSON string
response = ABSAResponse.model_validate_json(json_string)

# From dictionary
response = ABSAResponse.model_validate(dict_data)

    Field Constraints¶

    Polarity Values¶

    Valid polarity values: - "positive": Positive sentiment - "negative": Negative sentiment
    - "neutral": Neutral sentiment

    Score Range¶

    Scores must be between 0.0 and 1.0: - 0.0: No confidence - 0.5: Moderate confidence - 1.0: Maximum confidence

    String Constraints¶

    • id: Must be a valid UUID string
    • aspect: Should be descriptive (e.g., "camera quality", not "camera")
    • quote: Should be a relevant excerpt from the original text

    Usage Examples¶

    Creating Responses¶

    from app.schemas.response.absa_response import ABSAResponse, AspectAnalysisResponse
import uuid

# Create aspect analysis
aspect = AspectAnalysisResponse(
    aspect="camera quality",
    sentiment="positive",
    score=0.95,
    quote="camera quality is excellent"
)

# Create main response
response = ABSAResponse(
    id=str(uuid.uuid4()),
    polarity="neutral",
    score=0.5,
    aspects=[aspect],
    message="ABSA completed successfully.",
    text="The camera quality is excellent but the battery life is disappointing."
)

    Processing Responses¶

    # Access response data
print(f"Analysis ID: {response.id}")
print(f"Overall sentiment: {response.polarity}")
print(f"Confidence: {response.score}")

# Process aspects
for aspect in response.aspects:
    print(f"Aspect: {aspect.aspect}")
    print(f"Sentiment: {aspect.sentiment}")
    print(f"Confidence: {aspect.score}")
    print(f"Quote: {aspect.quote}")
    print("---")

    Error Responses¶

    # Create error response
error_response = ABSAResponse(
    id=str(uuid.uuid4()),
    polarity="neutral",
    score=0.0,
    aspects=[],
    message="Analysis failed: Invalid input",
    text=""
)

    Best Practices¶

    Response Creation¶

    1. Always use UUIDs: Generate unique IDs for each analysis
    2. Meaningful messages: Provide clear status messages
    3. Valid scores: Ensure scores are within 0.0-1.0 range
    4. Descriptive aspects: Use clear, descriptive aspect names

    Error Handling¶

    1. Graceful degradation: Provide fallback responses for errors
    2. Clear error messages: Include helpful error information
    3. Consistent structure: Maintain response structure even for errors

    Performance¶

    1. Efficient serialization: Use model_dump() for better performance
    2. Minimal data: Only include necessary fields in responses
    3. Caching: Cache frequently accessed response data

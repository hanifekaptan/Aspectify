    Ortam Kurulumu¶

    Aspectify'i farklı ortamlarda (geliştirme, test, üretim) kurmak için rehber.

    Ortam Türleri¶

    Geliştirme Ortamı (Development)¶

    Yerel geliştirme için optimize edilmiş ortam.

    Test Ortamı (Testing)¶

    Otomatik testler ve entegrasyon testleri için.

    Üretim Ortamı (Production)¶

    Canlı kullanıcılar için optimize edilmiş ortam.

    Geliştirme Ortamı¶

    Kurulum¶

    # Projeyi klonla
git clone https://github.com/your-username/aspectify.git
cd aspectify

# Sanal ortam oluştur
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate   # Windows

# Bağımlılıkları yükle
pip install -r requirements.txt

    Yapılandırma¶

    .env.development dosyası oluşturun:

    # Geliştirme ortamı ayarları
ENVIRONMENT=development
DEBUG=true
LOG_LEVEL=DEBUG

# Google API
GOOGLE_API_KEY=your_development_api_key

# Veritabanı (yerel)
DATABASE_URL=postgresql://aspectify:password@localhost:5432/aspectify_dev

# Pinecone (isteğe bağlı)
PINECONE_API_KEY=your_development_pinecone_key
PINECONE_ENVIRONMENT=us-west1-gcp
PINECONE_INDEX_NAME=aspectify-dev-index

# Uygulama ayarları
PROJECT_NAME=Aspectify-Dev
PROJECT_VERSION=1.0.0-dev

    Çalıştırma¶

    # Geliştirme sunucusu
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# Test çalıştırma
python main.py

# Test suite
pytest tests/

    Test Ortamı¶

    Kurulum¶

    # Test ortamı için ayrı sanal ortam
python -m venv .venv-test
source .venv-test/bin/activate

# Test bağımlılıkları
pip install -r requirements.txt
pip install pytest pytest-asyncio pytest-cov

    Yapılandırma¶

    .env.testing dosyası oluşturun:

    # Test ortamı ayarları
ENVIRONMENT=testing
DEBUG=false
LOG_LEVEL=WARNING

# Google API (test anahtarı)
GOOGLE_API_KEY=your_test_api_key

# Test veritabanı
DATABASE_URL=postgresql://aspectify:password@localhost:5432/aspectify_test

# Pinecone (test index)
PINECONE_API_KEY=your_test_pinecone_key
PINECONE_ENVIRONMENT=us-west1-gcp
PINECONE_INDEX_NAME=aspectify-test-index

# Uygulama ayarları
PROJECT_NAME=Aspectify-Test
PROJECT_VERSION=1.0.0-test

    Test Veritabanı¶

    -- Test veritabanı oluştur
CREATE DATABASE aspectify_test OWNER aspectify;

-- Test verilerini yükle
psql -U aspectify -d aspectify_test -f tests/fixtures/test_data.sql

    Test Çalıştırma¶

    # Tüm testler
pytest

# Belirli test dosyası
pytest tests/unit/test_absa_agent.py

# Coverage raporu
pytest --cov=app --cov-report=html

# Asenkron testler
pytest tests/integration/test_e2e_flow.py -v

    Üretim Ortamı¶

    Sunucu Gereksinimleri¶

    • OS: Ubuntu 20.04 LTS veya üzeri
    • CPU: 4 çekirdek, 3.0 GHz
    • RAM: 8 GB
    • Disk: 50 GB SSD
    • Ağ: 1 Gbps

    Kurulum¶

    # Sistem güncellemeleri
sudo apt update && sudo apt upgrade -y

# Gerekli paketler
sudo apt install -y python3.8 python3.8-venv python3-pip
sudo apt install -y postgresql postgresql-contrib
sudo apt install -y nginx certbot python3-certbot-nginx
sudo apt install -y git curl wget

# Uygulama kullanıcısı
sudo useradd -m -s /bin/bash aspectify
sudo usermod -aG sudo aspectify

    Uygulama Kurulumu¶

    # Uygulama dizini
sudo mkdir -p /opt/aspectify
sudo chown aspectify:aspectify /opt/aspectify

# Projeyi klonla
cd /opt/aspectify
sudo -u aspectify git clone https://github.com/your-username/aspectify.git .

# Sanal ortam
sudo -u aspectify python3.8 -m venv .venv
sudo -u aspectify .venv/bin/pip install -r requirements.txt

    Yapılandırma¶

    .env.production dosyası oluşturun:

    # Üretim ortamı ayarları
ENVIRONMENT=production
DEBUG=false
LOG_LEVEL=INFO

# Google API
GOOGLE_API_KEY=your_production_api_key

# Veritabanı
DATABASE_URL=postgresql://aspectify:strong_password@localhost:5432/aspectify_prod

# Pinecone
PINECONE_API_KEY=your_production_pinecone_key
PINECONE_ENVIRONMENT=us-west1-gcp
PINECONE_INDEX_NAME=aspectify-prod-index

# Uygulama ayarları
PROJECT_NAME=Aspectify
PROJECT_VERSION=1.0.0

# Güvenlik
SECRET_KEY=your_secret_key_here
ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com

    Veritabanı Kurulumu¶

    -- Üretim veritabanı
CREATE DATABASE aspectify_prod OWNER aspectify;

-- Güvenlik ayarları
ALTER USER aspectify PASSWORD 'strong_password';
REVOKE ALL ON SCHEMA public FROM PUBLIC;
GRANT ALL ON SCHEMA public TO aspectify;

    Systemd Servisi¶

    /etc/systemd/system/aspectify.service dosyası:

    [Unit]
Description=Aspectify API Service
After=network.target postgresql.service

[Service]
Type=exec
User=aspectify
Group=aspectify
WorkingDirectory=/opt/aspectify
Environment=PATH=/opt/aspectify/.venv/bin
ExecStart=/opt/aspectify/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

    Nginx Yapılandırması¶

    /etc/nginx/sites-available/aspectify dosyası:

    server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /static/ {
        alias /opt/aspectify/static/;
    }

    location /logs/ {
        alias /opt/aspectify/logs/;
        auth_basic "Restricted";
        auth_basic_user_file /etc/nginx/.htpasswd;
    }
}

    SSL Sertifikası¶

    # Let's Encrypt sertifikası
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

# Otomatik yenileme
sudo crontab -e
# 0 12 * * * /usr/bin/certbot renew --quiet

    Servisleri Başlatma¶

    # Servisleri etkinleştir
sudo systemctl enable aspectify
sudo systemctl enable nginx
sudo systemctl enable postgresql

# Servisleri başlat
sudo systemctl start aspectify
sudo systemctl start nginx
sudo systemctl start postgresql

# Durum kontrolü
sudo systemctl status aspectify
sudo systemctl status nginx
sudo systemctl status postgresql

    Docker Kurulumu¶

    Dockerfile¶

    FROM python:3.8-slim

WORKDIR /app

# Sistem bağımlılıkları
RUN apt-get update && apt-get install -y \
    gcc \
    g++ \
    libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# Python bağımlılıkları
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Uygulama kodu
COPY . .

# Kullanıcı oluştur
RUN useradd -m aspectify && chown -R aspectify:aspectify /app
USER aspectify

# Port
EXPOSE 8000

# Başlatma komutu
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]

    Docker Compose¶

    docker-compose.yml dosyası:

    version: '3.8'

services:
  app:
    build: .
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://aspectify:password@db:5432/aspectify
      - GOOGLE_API_KEY=${GOOGLE_API_KEY}
    depends_on:
      - db
    volumes:
      - ./logs:/app/logs

  db:
    image: postgres:13
    environment:
      - POSTGRES_USER=aspectify
      - POSTGRES_PASSWORD=password
      - POSTGRES_DB=aspectify
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl
    depends_on:
      - app

volumes:
  postgres_data:

    Docker Kullanımı¶

    # Geliştirme
docker-compose -f docker-compose.dev.yml up

# Üretim
docker-compose -f docker-compose.prod.yml up -d

# Logları görüntüle
docker-compose logs -f app

# Servisleri durdur
docker-compose down

    Monitoring ve Loglama¶

    Log Yönetimi¶

    # Log rotation
sudo nano /etc/logrotate.d/aspectify

/opt/aspectify/logs/*.log {
    daily
    missingok
    rotate 30
    compress
    delaycompress
    notifempty
    create 644 aspectify aspectify
    postrotate
        systemctl reload aspectify
    endscript
}

    Monitoring¶

    # System monitoring
sudo apt install htop iotop nethogs

# Application monitoring
pip install prometheus-client

# Health check endpoint
curl http://localhost:8000/health

    Backup¶

    # Veritabanı yedekleme
pg_dump -U aspectify -h localhost aspectify_prod > backup_$(date +%Y%m%d).sql

# Otomatik yedekleme
sudo crontab -e
# 0 2 * * * /usr/bin/pg_dump -U aspectify -h localhost aspectify_prod > /backup/aspectify_$(date +\%Y\%m\%d).sql

    Güvenlik¶

    Firewall¶

    # UFW yapılandırması
sudo ufw enable
sudo ufw allow ssh
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw deny 5432/tcp

    SSL/TLS¶

    # SSL sertifikası kontrolü
openssl x509 -in /etc/letsencrypt/live/yourdomain.com/cert.pem -text -noout

# SSL test
curl -I https://yourdomain.com

    Güvenlik Güncellemeleri¶

    # Otomatik güvenlik güncellemeleri
sudo apt install unattended-upgrades
sudo dpkg-reconfigure unattended-upgrades

    Sorun Giderme¶

    Yaygın Sorunlar¶

    1. Servis Başlamıyor¶

    # Logları kontrol et
sudo journalctl -u aspectify -f

# Servis durumunu kontrol et
sudo systemctl status aspectify

# Manuel başlatma
sudo -u aspectify /opt/aspectify/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000

    2. Veritabanı Bağlantı Hatası¶

    # PostgreSQL durumunu kontrol et
sudo systemctl status postgresql

# Bağlantıyı test et
psql -U aspectify -h localhost -d aspectify_prod

# Port kontrolü
sudo netstat -tlnp | grep 5432

    3. Nginx Hatası¶

    # Nginx yapılandırmasını test et
sudo nginx -t

# Nginx logları
sudo tail -f /var/log/nginx/error.log

# Servis durumunu kontrol et
sudo systemctl status nginx

    # Sistem kaynaklarını izle
htop

    En İyi Uygulamalar¶

    Güvenlik¶

    1. Güncellemeler: Sistem ve uygulamaları güncel tutun
    2. Güvenlik Duvarı: Gereksiz portları kapatın
    3. SSL: HTTPS kullanın
    4. Backup: Düzenli yedekleme yapın

    Performans¶

    1. Monitoring: Sistem metriklerini izleyin
    2. Logging: Uygun log seviyesi kullanın
    3. Caching: Sonuçları önbelleğe alın
    4. Load Balancing: Yüksek trafik için load balancer kullanın

    Bakım¶

    1. Log Rotation: Log dosyalarını döndürün
    2. Disk Space: Disk alanını izleyin
    3. Memory: Bellek kullanımını kontrol edin
    4. Updates: Düzenli güncelleme yapın
